# Validation Checks

## Structural Checks

- For a simplicial complex, verify every face of every simplex is present or generated by a backend that guarantees closure.
- For a cubical complex, verify grid dimension, cell indexing, and lower-star or bitmap convention.
- For a cell or chain complex, check boundary maps have compatible dimensions and `d_{n-1} d_n = 0`.
- For a filtration, verify every face appears no later than its coface.
- For a group route, record whether the group is finite, finitely presented, permutation, polycyclic, or only partially specified.
- For a polynomial route, record base field, characteristic, variables, term order, grading, and saturation choices.
- For triangulations or knots, record orientation, mirror, framing, component order, diagram encoding, polynomial normalization, reduced versus unreduced convention, and manifold hypotheses.

## Invariant Cross-Checks

- Euler characteristic equals the alternating sum of Betti numbers for finite complexes.
- Integral homology reduced modulo a prime should match field-homology expectations after accounting for torsion.
- `H_1` of a connected space should match the abelianization of the fundamental group when both are computed.
- Closed orientable manifold Betti numbers should satisfy Poincare duality when hypotheses are verified.
- Knot polynomial conventions should be compared against known examples such as the unknot, trefoil, figure-eight knot, and Hopf link.
- Hilbert series, Hilbert polynomial, dimension, and degree should be mutually consistent in graded algebraic examples.
- Primary decomposition should reconstruct the radical or ideal relation expected by the backend documentation.
- Numerical roots should be checked by residuals and certification where available.

## Known Examples

Use small examples before trusting a new route:

| Object | Expected signal |
| --- | --- |
| sphere `S^n` | Betti numbers in degrees 0 and n |
| torus `T^2` | Betti numbers 1, 2, 1 over a field |
| real projective plane | integral torsion in `H_1` |
| Klein bottle | nonorientable surface checks |
| cyclic group | abelianization and low-degree group homology sanity checks |
| unknot | trivial knot polynomial under chosen convention |
| trefoil | nontrivial Alexander/Jones polynomial and signature convention checks |
| Hopf link | nonzero linking number with component-order convention recorded |
| standard complete intersection | predictable Hilbert and Betti data |

## Evidence Report Checklist

Every result report should state:

- exact input artifact or source excerpt;
- representation used by the backend;
- coefficient ring or field;
- conventions that affect signs, gradings, order, or orientation;
- backend name and version;
- command or code path;
- raw log or durable output path;
- validation checks passed and failed;
- whether the result is exact, certified numerical, heuristic, partial, or failed.
